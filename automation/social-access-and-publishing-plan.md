# Matrix Reprogrammed — Facebook / YouTube / Video Automation Access Plan

## Important rule

Do not paste Facebook, Google, YouTube, ElevenLabs, or API passwords/tokens into ChatGPT chat.

Use platform permissions where possible, and store automation credentials only as GitHub Actions secrets, Netlify environment variables, or the secret vault inside the chosen automation platform.

---

# 1. What ChatGPT can and cannot control directly

## ChatGPT can help build

- Social post copy
- Weekly news posts
- Video scripts
- Captions
- Hashtags
- YouTube titles/descriptions/tags
- Rumble/Facebook/YouTube posting templates
- GitHub Actions automation scripts
- Site news/video page updates
- Content calendars
- Publishing workflows

## ChatGPT cannot directly do without integration

- Log into Facebook
- Accept Facebook Page access
- Log into YouTube Studio
- Upload to YouTube directly from the chat
- Post to Facebook directly from the chat
- Use private platform accounts without an approved connector/API/token setup

---

# 2. Best Access Model

Use two layers:

```text
ChatGPT/Codex -> GitHub repo -> GitHub Actions / automation platform -> Facebook + YouTube APIs
```

This keeps control with the owner and avoids password sharing.

---

# 3. Facebook Page Access Options

## Option A — Manual publishing / human approval

Use this first if you want safety and speed.

Workflow:

1. ChatGPT creates social posts, image prompts, video scripts, and captions.
2. Content is saved in the repo under `social/` or `news/`.
3. You copy/paste or schedule posts in Meta Business Suite.

Pros:

- Fastest
- No API approval headache
- Full control
- Low risk

Cons:

- Not fully automated

## Option B — Use Meta Business Suite access for a human operator

Use this if a real human assistant needs access.

Give them Page access inside Meta Business Suite / Business Settings. Assign only the minimum needed permissions.

Recommended permissions for a human social assistant:

- Content creation / posts
- Messages only if needed
- Insights only if needed
- No finance/admin ownership unless absolutely needed

Important:

ChatGPT itself has no Facebook profile or email to accept a Page invite.

## Option C — API automation

Use this for automatic posting.

Needed:

- Meta developer app
- Facebook Page ID
- Page access token
- Required publishing permissions, depending on the exact endpoint and account type
- Token stored as a GitHub Actions secret, not in the repo

Suggested GitHub secrets:

- `META_PAGE_ID`
- `META_PAGE_ACCESS_TOKEN`

Future automation script:

```text
weekly-news.json -> social caption -> video/image asset -> Meta Graph API publish
```

# 4. YouTube Channel Access Options

## Option A — YouTube Studio channel permissions

Use this for a real human manager/editor.

Recommended role:

- `Editor` if they only need upload/publish/content management
- `Manager` only if they also need permission management or deeper channel control

Important:

ChatGPT itself has no Google account to accept a YouTube Studio invite.

## Option B — YouTube Data API automation

Use this for automated video uploads.

Needed:

- Google Cloud project
- YouTube Data API enabled
- OAuth consent configured
- OAuth refresh token generated by the channel owner
- Client ID / client secret / refresh token stored as GitHub Actions secrets

Suggested GitHub secrets:

- `YOUTUBE_CLIENT_ID`
- `YOUTUBE_CLIENT_SECRET`
- `YOUTUBE_REFRESH_TOKEN`
- `YOUTUBE_CHANNEL_ID`

Warning:

Uploads from unverified API projects may be restricted to private visibility until the project passes Google API compliance/audit requirements.

# 5. ElevenLabs / AI Video Automation

## Needed secrets

- `ELEVENLABS_API_KEY`
- `ELEVENLABS_VOICE_ID`

## Possible video workflow

```text
Intel Desk item -> script -> ElevenLabs voiceover -> image/video generation/render -> captions -> upload to site -> optional YouTube upload -> optional Facebook post
```

## Safer initial workflow

Start with semi-automation:

1. ChatGPT creates weekly scripts and captions.
2. ElevenLabs voiceover is generated manually or by workflow.
3. Rendered video is reviewed.
4. Human approves posting.
5. Later, automate publishing after quality is proven.

---

# 6. Recommended Rollout

## Phase 1 — Content automation inside repo

- Generate weekly Intel Desk drops
- Generate social captions
- Generate YouTube/Rumble descriptions
- Generate video scripts
- Update `news.html` and `videos.html`

## Phase 2 — Human approval publishing

- You review generated posts/videos
- Publish manually in Meta Business Suite / YouTube Studio

## Phase 3 — API automation

- Add API secrets to GitHub Actions
- Upload videos to YouTube from workflow
- Post links/captions to Facebook Page
- Add logging and failure reports

## Phase 4 — Full loop

```text
weekly news scan -> sourced post -> site update -> video render -> YouTube upload -> Facebook post -> email/social pack
```

# 7. Immediate User Steps

## Facebook

1. Confirm whether the page is controlled through Meta Business Suite.
2. Decide manual approval first or API automation.
3. If API automation, create/identify the Meta developer app and Page ID.
4. Store Page credentials as GitHub secrets when ready.

## YouTube

1. Create/confirm the new YouTube channel.
2. Customize name, handle, banner, description, and links.
3. For human access: invite an Editor/Manager in YouTube Studio.
4. For automation: create a Google Cloud project, enable YouTube Data API, generate OAuth refresh token, then store secrets in GitHub.

## ElevenLabs

1. Choose the voice.
2. Get the voice ID.
3. Store the API key and voice ID as GitHub secrets.

# 8. Safety / Brand Rules

- Keep claims evidence-labeled.
- Never post fake client lists.
- Never name private victims.
- Use source links for public-record claims.
- Use satire carefully and label it clearly.
- Avoid unsupported allegations against living people.
- Every news post should lead to a source, a shelf, a book, or the Black File.
