# Epstein Network Deep Dive Matrix

Updated: 2026-06-29

## Evidence Boundary
This matrix tracks public-record mentions, court records, congressional releases, reported emails, flight/contact records, institutional roles, properties, logistics, and chronology. A name, flight, contact, meeting, location, or email is not proof of criminal conduct. Use evidenceClass, sourceType, and claimBoundary before making any claim.

## Source Hierarchy
- Court docket / sworn testimony / conviction / plea
- Congressional release / official archive / DOJ disclosure
- Authenticated email or document archive
- Flight log / contact book / calendar / visitor record
- Major dated reporting that links to underlying documents
- Open question / unverified claim / speculation quarantine

## Private Jet Tracker Rules
- Track aircraft tail number only when sourced from flight logs or official/reputable document archives.
- For each flight, store date, origin, destination, passenger names as written, source, confidence, and evidence boundary.
- Never state that a passenger committed a crime because of a flight. Use association/travel language unless court records support more.
- Flag repeated corridors and same-window overlaps for research, not as proof.
- Separate verified flight logs from social-media flight claims and sighting claims.

## Deep-State / Institutional Overlap Framing Rules
- Use 'institutional overlap', 'state-power overlap', 'intelligence-adjacent', 'prosecutorial failure', 'political-access lane', or 'elite-protection question' instead of presenting an unsupported 'deep state' conclusion.
- A deep-state score must be an editorial research signal only, based on public role, institutional power, record proximity, secrecy/redaction relevance, and legal/disclosure involvement.
- Never use the score as proof of guilt.
